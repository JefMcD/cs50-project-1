regular expressions are used to operate on strings

Metacharacters usedin regex => . ^ $ * + ? { } [ ] \ | ( )
character classes are groupes using []
[abc] is characters a,b,c
[a-c] is also characters a,b,c
[abc$] is characters a,b,c,$ - metacharacters lose their special power inside a class groupe
[^abc] is not a,b,c - anywhere after the first position it loses its special power

The \ metacharacters
\$, \\, \? etc - escapes the metacharacters special powers so it can be matched as an ordinary character

\d - any decimal digit; this is equivalent to the class [0-9].
\D - Matches any non-digit character; this is equivalent to the class [^0-9].

\s - Matches any whitespace character; this is equivalent to the class [ \t\n\r\f\v].
\S - Matches any non-whitespace character; this is equivalent to the class [^ \t\n\r\f\v].

\w - Matches any alphanumeric character; this is equivalent to the class [a-zA-Z0-9_].
\W - Matches any non-alphanumeric character; this is equivalent to the class [^a-zA-Z0-9_].

These sequences are often included inside a class
[\s,.'"] - matches any whitespace , . ' " [\dab] - matched any number also a and b [\w01] - matches any character also 0 and 1

Metacharacter => .
The . matches anything except a newline character

This can be altered to include anything including the newline by using
re.DOTALL
then the . will match the newline as well

the * + ? metacharacter

[ca*t] - matches 'ct' 'cat' 'caaat' the a is matched 0 or more timesetc
[ca+t] - matches c then at least one a then t 'cat' 'caat' etc
[ca?t] - matches 'cat' 'ct' makes the a an optional character 0 or 1 times

{m,n} quantifier
[ca{1,3}n] - matches 'cat', 'caat', 'caaat'
matches at least m and at most n occurences


ommittinf m or n
{m,} - n defaults to infinity
{,n} - m defaults to 0
